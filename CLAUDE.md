# CLAUDE.md — Prod or Pretend Session Instructions

## What This Repo Is

`OMNIXIS-prod-or-pretend` is the planning and (eventually) code repository for **Prod or Pretend** — a constructive quality mirror for tech hype on LinkedIn and X. Part of the Neckarshore AI product portfolio.

## Product Context

- **What:** Analyzes hyped tech posts against production-ready standards. Dual-path: repo exists (technical deep analysis) vs. no repo (qualitative mirror against consultant framework).
- **Target audience:** Decision-makers (CTOs, founders, team leads) — NOT developers.
- **Tone:** Constructive-critical. Mirror reality, don't attack. Solution-oriented.
- **Channels:** LinkedIn (neckarshore.ai, human-in-loop) + X (Prod or Pretend brand, looser tone).
- **Monetization:** Acquisition channel for consulting mandates, not a paywall product.

## How to Work in This Repo

### Documentation
- All docs in `docs/`. The PRD is the source of truth for product decisions.
- `input/` contains raw brainstorming session exports — reference material, not to be edited.
- `backlog/ideas.md` captures deferred ideas.

### Style
- English for all documents.
- No emoji in documents.
- Follow Neckarshore AI formatting conventions (same as OMNIXIS-planning).

### Key Decisions
- See `docs/decisions.md` for all decisions with status (Decided/Open/Deferred).
- Check decisions before making changes that could conflict.
